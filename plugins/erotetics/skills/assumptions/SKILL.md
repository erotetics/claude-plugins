# Assumption analysis (Surface → Examine → Challenge → Reframe)

The method, from Erotetic Arts (erotetics.com): expose the load-bearing
assumptions behind a claim or decision and converge on a question that can
actually be settled. The output is not more analysis — it is clearer closure.

## 0. Establish the subject

- If arguments were given ("$ARGUMENTS"), analyse that.
- Otherwise use the content already in context (a file, plan, diff, or pasted
  text the user pointed at).
- If a browser-capture MCP tool is available (e.g. a Clip bridge exposing
  `get_latest_capture`), a fresh capture may be the subject — a marked region
  or selection in it is the focus claim; the rest is context.
- If nothing is identifiable, ask for the subject — never analyse a guess.

## 1. SURFACE — name what is taken for granted

Enumerate presuppositions, stated and unstated. Hunt in these categories:

- **Factual givens** — claims treated as true without support
- **Smuggled definitions** — contested terms used as if settled ("growth",
  "safe", "the user")
- **Causal links** — X→Y treated as mechanism when it is correlation or hope
- **The question behind the answer** — what question is this content
  answering? Was that ever the right question?
- **Success criteria** — what "working" is assumed to mean, and for whom
- **Actors & incentives** — who is assumed to act, cooperate, or stay still
- **Temporal load** — what must *remain* true for this to keep holding

## 2. EXAMINE — find the load-bearing walls

For each surfaced assumption, classify in a table:

| Assumption | Load-bearing? (what breaks if false) | Evidence status (verified / merely assumed / contradicted) |

Then list EVERY remaining surfaced assumption in a one-line "Noted, not
load-bearing" appendix with a clause of why it doesn't gate the decision —
never silently drop one. (Evals showed silent discarding costs real coverage:
convergence must not purchase focus with blindness.)

## 3. CHALLENGE — strongest honest counter-case

For the top 2–4 load-bearing assumptions only:
- The best case that it is false (steelman, not strawman)
- What evidence would settle it either way
- The **cheapest test** that would produce that evidence

## 4. REFRAME — the question to settle

Restate the actual question the subject should be answering — often narrower
and more decidable than the one it started with. One or two candidate
reframes, each phrased so it CAN be settled (by evidence, by a decision-maker,
by a test). End with: **"The question to settle: …"**

## Grounding rules

- Quote the subject verbatim when attributing an assumption to it; from a web
  capture, cite selectors/tiles so the pointer is checkable.
- Base every surfaced assumption on the provided content — do not invent
  positions the author never took.
- Keep it tight: a decision-maker should act on this in one read.
